REQUERIMIENTOS FUNCIONALES – PROYECTO KANBAN

1. El sistema debe permitir crear un nuevo proyecto con un nombre definido por el usuario.
2. El sistema debe permitir agregar tareas a un proyecto, indicando su nombre.
3. El sistema debe permitir mostrar todas las tareas del proyecto, mostrando su nombre y estado.
4. El sistema debe permitir cambiar el estado de una tarea (Pendiente, En progreso, Completada).
5. El sistema debe permitir eliminar tareas existentes.
6. El sistema debe organizar las tareas dentro del proyecto usando colecciones ArrayList.
7. El sistema debe implementar clases, objetos, atributos y métodos para modelar el proyecto y las tareas.
8. El sistema debe aplicar herencia entre clases (TareaBase, TareaNormal, TareaUrgente).
9. El sistema debe usar encapsulamiento y visibilidad adecuada para sus atributos y métodos.
10. El sistema debe usar una interfaz (Movible) para definir comportamientos comunes entre tipos de tareas.
11. El sistema debe mostrar un menú interactivo por consola que permita navegar entre las opciones.
12. El sistema debe estar preparado para implementar gestión de excepciones y lectura/escritura de archivos en versiones futuras.

REQUERIMIENTOS NO FUNCIONALES

1. El código debe ser modular, legible y documentado.
2. Las clases deben mantener una relación coherente (asociación, agregación y herencia).
3. El sistema debe ser fácil de ampliar para futuras versiones (por ejemplo, añadir persistencia o interfaz gráfica).
4. Los nombres de clases, métodos y variables deben seguir convenciones claras y consistentes.
